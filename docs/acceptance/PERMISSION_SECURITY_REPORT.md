# 权限与安全验收报告

> 状态：PASS
> 范围：认证、五级角色、数据范围、详情、导出、预警、飞书目标群、CSRF、越级与敏感信息。

## 权限矩阵结果

- L5 开发者：ALL、全部当前及未来直播间、数据库入口。
- L4 管理员：全部正式直播间与平台管理能力，不含数据库入口。
- L3 运营负责人：全部正式直播间、数据源、同步、ROI 目标、预警规则和审计，不含用户、角色、系统密钥。
- L2 直播主管：全部正式直播间、经营查看、导出和预警处置。
- L1 项目 PM：仅所属项目直播间的经营查看、导出和预警查看。

详细矩阵与角色配置见 `docs/PERMISSIONS.md`。

## 越权测试

- 未登录访问受保护接口返回 HTTP 401；无对应权限点返回 HTTP 403。
- 客户端显式请求范围外直播间返回 HTTP 403。
- 管理员不能创建、修改凭据、停用、删除或授予开发者/同级管理员角色。
- 管理员不能把 L3/L2/L1 角色配置为隐藏管理员；权限点最低等级由服务端验证。
- 运营负责人可访问数据源和审计，不能访问系统设置与用户权限总览。
- 当前账号不能修改自身角色/启停状态或删除自身；最后一个开发者不能被移除。

## 安全扫描

- Ruff、mypy、ESLint、TypeScript、Prettier、CSRF 路由扫描全部通过。
- 189 个后端测试、68 个前端单测和 6 个 Playwright E2E 通过；领域与服务覆盖率 85.89%。
- 密码、密码哈希、Token、Secret 和 Webhook 不进入权限 API 响应或审计明文。
- `make.cmd verify-production` 验证强密钥、关闭开发旁路、无 fixture 写入和 Docker 构建路径。

## 结论

五级角色、按直播间数据范围和防越级授权均由服务端强制执行，满足本阶段上线条件。本机没有 Docker CLI，容器运行态仍需在具备 Docker 的目标服务器执行一次实际启动验收。
